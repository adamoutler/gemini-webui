Technical Nuances and Operational Capabilities Identified:

1. Architectural & Operational Nuances
- Storage Failover Logic: The app implements a 'Fail-Closed' storage paradigm. If the primary /data volume is read-only, it automatically falls back to /tmp/gemini-data. If both are unavailable, it logs a critical warning but remains functional in a limited state.
- Automatic Permission Management: On startup, the app detects if mounted volumes are root-owned (common with Docker mounts) and recursively chowns them to the node user (UID 1000).
- Background Session Preloader: A background task warms the session cache on startup for all configured hosts, improving launcher responsiveness.
- PTY Lifecycle Management: A dedicated 'reaper' task (cleanup_orphaned_ptys) monitors disconnected sessions and kills PTY processes that have been orphaned for more than 60 seconds.
- Proxy Compatibility: Uses ProxyFix middleware to correctly handle X-Forwarded-For, X-Forwarded-Proto, and other proxy headers, ensuring correct IP and protocol detection behind load balancers.

2. Mobile & UX Optimizations
- Android Keyboard Workaround: Implements a 'Single Character Buffer Hack' for the terminal input. It disables autocorrect/autocapitalize and clears the textarea immediately after each keystroke to prevent mobile OS interference with terminal input.
- Visual Viewport API Integration: Dynamically adjusts the application height and terminal fit when the mobile on-screen keyboard is toggled, preventing UI breakage.
- ANSI Navigation Overlay: The mobile control bar sends raw ANSI escape sequences (\x1b[A, etc.) directly to the PTY, providing reliable arrow key functionality on mobile devices.
- Intelligent Link Handling: A custom LinkProvider for xterm.js specifically handles multi-line wrapped URLs, allowing users to click links that span multiple terminal lines.

3. Security Hardening
- LDAP Injection Protection: Input sanitization in auth_ldap.py strips characters like (, ), and * from usernames before passing them to LDAP queries.
- Granular CSP: Flask-Talisman is configured with a strict Content Security Policy that explicitly whitelists specific CDNs (jsdelivr, ccdnjs, unpkg) for scripts and styles.
- Secure Session Configuration: Cookies are hardened with HttpOnly and SameSite=Lax settings.
- SSH Target Validation: A regex-based validator ensures SSH targets conform to safe user@host formats before execution.

4. SSH & Connectivity Nuances
- Login Shell Parity: Remote commands are wrapped in bash -l -c to ensure that remote .profile or .bash_profile paths are respected.
- RO Known-Hosts Handling: If the SSH directory is read-only, the app falls back to using /dev/null for UserKnownHostsFile to prevent SSH failures.
- Remote Environment Passthrough: Automatically exports TERM, COLORTERM, and FORCE_COLOR to remote sessions to ensure terminal fidelity matches the local experience.

5. Undocumented Environment Variables
- SKIP_MONKEY_PATCH: Disables Eventlet monkey-patching.
- GEMINI_BIN: Custom path to the Gemini executable.
- LDAP_FALLBACK_DOMAIN: Domain used for direct BIND when no service account is provided.
- BYPASS_AUTH_FOR_TESTING: Disables all authentication for development.
- SKIP_PRELOADER: Disables the startup session cache warming.
- FLASK_USE_RELOADER: Controls the Flask auto-reload feature.
