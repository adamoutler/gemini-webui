# jenkins Module

## Purpose

The `jenkins/` directory contains automation logic that provides a feedback loop for developers and AI agents after a `git push`. The core file, `wait-for-receipt.sh`, monitors build progress by watching a temporary log file (`/tmp/jenkins-receipt-gemini-webui.log`) generated by Jenkins. Upon success, it syncs the local repository to GitHub. Upon failure, it provides actionable instructions for AI agents to troubleshoot. It is deeply integrated into the project's dev environment to ensure immediate feedback.

## Internal Dependencies

- **Jenkins Pipeline Logs**: Depends on the root-level `Jenkinsfile` writing specific status signals to `/tmp/jenkins-receipt-gemini-webui.log`.

## External Dependencies

- **`setup_dev.sh`**: The project's development setup script hooks into the `git push` command, automatically calling `jenkins/wait-for-receipt.sh` to provide an inline CI/CD receipt.
- **Developer Workflow**: Relied upon by developers and autonomous agents to confirm the health of their commits before moving on to subsequent tasks.
