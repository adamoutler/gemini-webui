# tests/utils Module

## Purpose

The `tests/utils` directory provides shared helper functions and developer-focused utility scripts that facilitate end-to-end testing and task management. It primarily serves two functions:

- **Mobile Interaction Simulation**: Contains functions that allow end-to-end tests to simulate complex mobile-specific interactions (like autocorrect and spacebar trackpad navigation) within the browser context.
- **Kanban Utilities**: Provides scripts for developers to query and list task details and descriptions directly from the project's Kanban tool outputs for easier status tracking.

## Internal Dependencies

- **Python Standard Library**: Uses `json` for processing task data and `time` for managing execution pacing in browser simulations.
- **Playwright Context**: Interacts with the `page` object's JavaScript environment to simulate mobile input events and selection changes.
- **Local Tool Outputs**: Depends on JSON-formatted output files generated by the `mcp_kanban` tool, located within the project's temporary session directories.

## External Dependencies

- **End-to-End Test Suite**: Imported and used by `tests/e2e/test_mobile_input_automation.py` and `tests/e2e/test_mobile_cursor_and_autopunct.py` to verify mobile viewport behaviors.
- **Project Infrastructure**: Mentioned in `tests/GEMINI.md` as a key repository for test-related utility code.
- **Developer Workflow**: Relied upon by developers as command-line utilities for task management and status reporting.
